<objective>
Make the `fleet_analytics` package importable at runtime and make output regeneration a clean no-op on Windows.

Purpose: Two of the three SHIP-01 blockers are foundational and must land first. (1) `pyproject.toml` has no `[build-system]`, so `uv` treats the project as non-packaged — `uv run python -m fleet_analytics.*` fails with `ModuleNotFoundError` (live-verified in research). pytest's `pythonpath=["src"]` only fixes imports under pytest, not at runtime. (2) Without a `.gitattributes`, DuckDB's LF-writing `COPY` exporters make every regenerated CSV show as "modified" on Windows even when values are byte-identical, which would break the "regeneration is a clean diff" ship verification in Plan 03.

This plan is pure packaging config — no Python logic, no new SQL surface.

Output: Edited `pyproject.toml` (importable package), new `.gitattributes`, updated `.gitignore`, and a synced `.venv` where `import fleet_analytics` works at runtime.
</objective>
