<objective>
Create the single one-command orchestrator that runs the entire data pipeline end-to-end, and a smoke test that proves it works.

Purpose: SHIP-01 Success Criterion 1 requires "the full pipeline runs as a single one-command invocation." Today no such orchestrator exists — `export.write_gold` is only ever called from tests, and there is no `pipeline.py`. CLAUDE.md already documents `uv run python -m fleet_analytics.pipeline` as the canonical invocation, so the module name and path are fixed. This plan wires the existing, already-tested builder functions into one `main()` on a single DuckDB connection, in the dependency order encoded in `tests/conftest.py`. It introduces NO new SQL and NO new interpolation — pure orchestration of existing builders (security invariant T-6-01).

Output: `src/fleet_analytics/pipeline.py` (full-chain `main()` + `__main__`) and `tests/test_pipeline.py` (integration smoke test).
